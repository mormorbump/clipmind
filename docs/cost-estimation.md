# ClipMind — コスト試算

> 個人学習プロジェクトの予算は **月 $30 以内**。
> 1時間動画の処理コストを $1.0 以下に抑えることが目標。

---

## 1. 前提

- 動画: 1時間（60分）
- フレーム抽出: シーンカット検出で **1分あたり平均 5 フレーム = 300 フレーム**
- 字幕: 英語で平均 **9000 単語（≒ 12000 tokens）**
- クエリ: 1動画あたり 30 クエリ想定

---

## 2. Ingest コスト（1時間動画あたり）

### 2.1 ローカルで処理する部分（API費用 $0）

| コンポーネント | 処理 | 推定時間 |
|---|---|---|
| yt-dlp | 取得（対応動画のみ） | 〜1分 |
| OpenCV | シーンカット + キーフレーム | 〜2分 |
| Whisper large-v3 (GPU) | 書き起こし | 〜3分 |
| Whisper base (CPU) | 書き起こし（軽量版） | 〜15分 |
| YOLOv8n (CPU) | 物体検知 | 〜2分 |
| Embedding (text-embedding-3-small) | 300 segments | 〜1秒 |

### 2.2 外部API費用

| 用途 | モデル | 単価 | 計算 | 金額 |
|---|---|---|---|---|
| フレームキャプション | **GPT-4o-mini** (1画像入力) | $0.000150/1k tokens in, $0.000600/1k out | 300枚 × (〜1k + 100 out) | **~$0.06** |
| フレームキャプション（Sonnet 版） | Claude Sonnet 4.6 | $3/M in, $15/M out | 300枚 × (〜1500 + 100 out) | ~$1.80 |
| Embedding | text-embedding-3-small | $0.020/M tokens | 12k tokens | **$0.0003** |
| 要約（fuse用） | Claude Haiku 4.5 | $0.80/M in, $4/M out | 15k in + 2k out | **~$0.02** |

**戦略**: デフォルトで **GPT-4o-mini でキャプション** → **Haiku で要約**
- 合計 Ingest コスト: **約 $0.10 / 1時間動画** ✓

Claude Vision（Sonnet）は以下の場合のみ使う（ユーザー指定時）:
- 高品質なアーカイブ用キャプションが必要な時
- その場合は ~$1.80 / 1時間動画

---

## 3. Query コスト（1クエリあたり）

| 処理 | モデル | トークン見積 | 金額 |
|---|---|---|---|
| Embedding（クエリ） | text-embedding-3-small | 50 tokens | ~$0.000001 |
| Rerank（Cohere） | rerank-english-v3.0 | 20件 | $0.001 / クエリ |
| LLM 生成 | Claude Sonnet 4.6 | 3k in + 500 out | $3×0.003 + $15×0.0005 = **~$0.017** |

**1 クエリあたり**: **~$0.018**

30 クエリで **$0.54**。

---

## 4. 月次コスト見込み

### シナリオ A: 学習開発フェーズ
- 動画 Ingest: 週 3 本（計 12 本/月）× 1時間
- クエリ: 1 本あたり 50 クエリ（合計 600 クエリ/月）

| 項目 | 金額 |
|---|---|
| Ingest × 12 | $0.10 × 12 = $1.20 |
| Query × 600 | $0.018 × 600 = $10.80 |
| 評価バッチ（Ragas等） | 〜$5.00 |
| バッファ | $3.00 |
| **合計** | **~$20** ✓ 予算内 |

### シナリオ B: デモ公開フェーズ
- Ingest: 30 本/月
- クエリ: 合計 2000/月
- コスト: **~$50** → 超過するため、Rate Limit / キャッシュ必要

## 5. コスト削減テクニック

### 5.1 モデル階層化
```
質問分類:
  単純 → Haiku 4.5       ($0.80/M)
  標準 → GPT-4o-mini     ($0.15/M)
  高難度 → Sonnet 4.6    ($3/M)
```

### 5.2 Prompt Caching
- Anthropic の Prompt Caching（ベース system prompt + 動画メタデータをキャッシュ）で 90% コスト削減
- `cache_control: {"type": "ephemeral"}` を使う

### 5.3 Embedding Caching
- 同じクエリが来たら embedding を再計算しない（Redis に 24h キャッシュ）

### 5.4 Rerank の条件付き実行
- Dense 検索の top-1 スコアが高ければ rerank をスキップ

### 5.5 フレーム選別の強化
- 300 → 100 枚に絞れるなら、キャプションコストが 1/3 に
