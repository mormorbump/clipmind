# ClipMind — 要件定義

## 1. プロジェクトゴール

動画内容を自然言語で検索・質問できるRAGシステムを構築し、
以下の実装経験を個人で一貫して積むこと:

- LLMアプリケーション実装
- LangChain / LlamaIndex エージェント開発
- API設計・外部API統合
- 映像解析（OpenCV / YOLO）＋マルチモーダルAI
- ベクトルDBを用いたRAG
- LangGraph マルチエージェント・オーケストレーション

---

## 2. ユーザーストーリー

### US-01 動画を投入して解析できる
**As a** 研究者 / コンテンツ管理者
**I want** 動画ファイル or URL を投入するだけで自動解析されること
**So that** 後で検索できる状態になる

**受け入れ基準**:
- ローカル動画ファイル（<2GB、mp4/mov/mkv/webm）を投入できる
- Creative Commons動画のURLを投入できる（YouTube URLは既定で無効）
- 投入後、処理進捗が WebSocket でリアルタイム取得できる
- 処理完了時に summary と timeline が生成されている

### US-02 自然言語で動画内容を検索できる
**As a** ユーザー
**I want** 「人物Aが登場するシーン」のような自然言語で検索
**So that** タイムスタンプと該当フレームが返る

**受け入れ基準**:
- 検索結果は関連度順に最大 10 件、各件に timestamp とサムネイル URL が含まれる
- Recall@5 が 評価セットで 0.7 以上
- レスポンスタイム p95 が 3 秒以下（LLM呼び出しは除く純粋検索部）

### US-03 動画について質問・対話できる
**As a** ユーザー
**I want** 「この動画の結論は何？」のような質問を対話形式で投げたい
**So that** 長時間動画でも要点を素早く把握できる

**受け入れ基準**:
- 回答にはタイムスタンプ引用が必ず付く（ハルシネーション抑制）
- 会話履歴が維持される
- ストリーミング応答（WebSocket）

### US-04 複数動画を横断検索できる
**As a** ユーザー
**I want** 複数の動画を一度に横断して検索したい
**So that** シリーズもの・コレクションを統合的に扱える

**受け入れ基準**:
- 1 検索で最大 100 本の動画を横断検索できる
- 結果に video_id と timestamp が必ず含まれる

## 4. 非機能要件

### 4.1 パフォーマンス
- 10分動画の Ingest: 10分以内（GPUあり環境）
- 検索 p95: 3秒以下（LLM 生成除く）
- 対話 TTFT（最初のトークンまで）: 2秒以下

### 4.2 信頼性
- Ingest 失敗時は最後のチェックポイントから再開可能
- 外部 LLM API 失敗時は別プロバイダへフォールバック

### 4.3 セキュリティ
- アップロードサイズ上限: 2GB
- 拡張子/MIME チェック
- SSRF 対策（URL取り込み時のホスト検証）
- APIキーは環境変数管理

### 4.4 コスト
- 1時間動画の Ingest コスト目標: $1.0 以下
- 詳細は `docs/cost-estimation.md`

### 4.5 観測性
- 全 LLM 呼び出しに trace_id
- メトリクス: ingest_duration, llm_tokens_total, rag_recall

### 4.6 保守性
- Python 3.11+, type hints 100%
- ruff + mypy を CI で強制
- テストカバレッジ: 70% 以上（ロジック層）

## 5. スコープ外（Non-Goals）

- マルチテナント・ユーザー管理（シングルユーザー前提）
- 動画編集機能
- ライブストリーム対応
- YouTube 動画の直接ダウンロード（ToS上実装しない）
- モバイルアプリ

---

## 6. 制約

- 開発環境: macOS（Apple Silicon）— YOLO/Whisper は MPS 対応が必須
- 本番環境想定: Linux + NVIDIA GPU（もしくは CPU-only + 小型モデル）
- 予算: 個人学習のため月 $30 以内に収める（API コスト）
